1-1. 脱・一発生成の罠

Mindset

今の自分を「拡張」するスキル

AIエンジニアに転職する必要はありません。今のあなたの専門知識や業務に、AIという最強の武器を掛け合わせることで、インプットの質を飛躍させ、あなたの可能性を劇的に広げることがLarryの目的です。

AIは「優秀だが、空気が読めない新入社員」

現実の仕事で、新入社員に「いい感じの企画書作っといて！」と一言だけ伝えて、一発で完璧なものが上がってくるでしょうか？ AIも同じです。

アマチュアの思考

  • ・1回のプロンプトで全てを解決しようとする
  • ・長文で複雑すぎる指示を最初から投げる

プロ（Larry流）の思考

  • ・まずは「60点の叩き台」を最速で作らせる
  • ・ラリー（往復）を通じて100点に育て上げる
Preparation

AIのアイデアを映し出す「鏡」を用意する

プログラミングをゼロから学ぶ必要は一切ありません。しかし、AIが圧倒的な質で出力してくれたコード（設計図）を、自分の目で見て確認するための「砂場（プレビュー環境）」は絶対に必要です。

1. VS Code と Live Server の導入

公式サイトからVS Codeをインストールし、拡張機能から「Live Server」を追加するだけです。これがあなたの新しい作業台になります。

2. 「Go Live」で形にする

フォルダを開き、AIからコピペした index.html を保存したら、右下の「Go Live」をクリック。これだけで、文字列が本物のWeb画面に変換されます。※フォルダごと開かないとボタンが出ないので注意！

Anti-Patterns

初心者が必ず引っかかる「AIの罠」

AIに指示を出す際、多くの人が「見た目」だけを整えさせて満足してしまいます。ここでは、AIが吐き出す「使えないゴミコード」を防ぐための具体的なツッコミ（軌道修正）の型を学びます。

罠①：「見た目」だけを整えさせる

初心者のNGプロンプト

「検索しやすいUI案を作って」

AIは「見た目」を整えるのが得意なため、中身がスッカラカンな箱だけを出してきます。

プロの回避策（ラリー）

「実際に動かすための『ノウハウDB（裏側の仕組み）』を組み込んで作り直して。」

枠組みができた段階で「裏側の仕組みの欠落」を鋭く指摘します。

Step 3-1

まずは「砂場（Sandbox）」を作らせる

システム構築の第一歩です。ここで絶対にやってはいけないのは「最初から100個の要望を詰め込むこと」です。AIが混乱し、途中で処理落ちを起こします。
第1球の目的は、AIに「今から何を作るのか」の全体像を理解させることです。

あなたの番です（第1球を投げる）

Prompt_Step1.txt 
社内の日報入力ツールを作りたいです。
HTMLとCSSでシンプルな枠組みを作ってください。

必要な入力項目は以下の3つです。
・日付
・名前
・業務内容

期待される実行結果


コピーしてシミュレーションを実行
シンプルな枠組みが出力されます

AIは指示通りに項目を作りますが、未入力でも送信できてしまう「使えない箱」が完成します。これで準備完了です！

Step 3-2

「使えない箱」を実務ツールに進化させる

第1球で土台ができました。プロのラリーはここからです。実運用に耐えうるシステムにするための「第2球」を投げ込みます。

「未入力エラーを防ぐ」「Tailwind CSSを指定する」など、具体的な要件を指示します。

あなたの番です（第2球を投げる）

Prompt_Step2.txt 
先ほどのコードは枠組みとしてはOKですが、実運用には耐えません。
以下の3点を追加して作り直してください。

1. 【エラー防止】未入力で送信した場合、赤文字でエラーを出す。
2. 【UI改善】Tailwind CSSを用いてモダンで綺麗なデザインにする。
3. 【項目追加】「明日の予定」欄を追加する。

期待される実行結果


コピーしてシミュレーションを実行
AIが神コードを出力します！

綺麗なUIと、JavaScriptの入力チェック機能が追加され、一気にプロ級のシステムに変貌します。

Step 3-3

神は細部に宿る。UXの最終調整

第2球でシステムの80%が完成しました。最後の第3球では、「実際に使う人がどう感じるか」というUXの観点で微調整を行います。ここで前回の出力を褒めることで、AIの「先祖返り」を防ぐのがコツです。

あなたの番です（最後の仕上げ）

Prompt_Step3.txt 
素晴らしいです。機能もデザインも良くなりました。
最後に、UX（使い勝手）向上のため以下の2点を微調整してください。

1. フォーム全体にもう少しゆとり（広めの余白）を持たせてください。
2. 入力内容をリセットする「クリアボタン」を追加してください。

変更を加えた完全なコードを出力してください。

期待される実行結果


コピーしてシミュレーションを実行
プロジェクト完了！

誰に出しても恥ずかしくない完璧なWebシステムが完成しました。これがラリーの力です！

The Output

インプットを「社内評価」に変換する

Module 4までで、あなたはAIからプロ顔負けのシステムや企画書を引き出す「インプット（ラリー）の質」を極めました。
しかし、自分のPCの中で動いているだけでは誰も評価してくれません。ここからは、その質の高い成果物を「上司やクライアントにドヤれる形（URLやPDF）」にして外へ届ける、アウトプットのスキルを学びます。

GitHub Pages

一瞬で世界へ繋ぐ「URL化」の魔法

「GitHub Pages」を使えば、ドラッグ＆ドロップだけであなたの作った成果物が全世界に公開されます。「これ、AIで作ってみたので触ってみてください」とURLをチャットで送るだけで、周りの見る目は確実に変わります。

  • 1. リポジトリ作成： GitHubで新しい箱（Public）を作る。
  • 2. ドロップ＆コミット： 作成した index.html などを直接放り込んで保存。
  • 3. Pagesをオンにする： 設定からBranchを「main」にするだけ。数分でURLが発行されます。
PDF Print

完璧な企画書に仕上げる「PDF調教術」

Web画面をそのまま印刷（Ctrl+P）すると、背景色が消えたりレイアウトが崩れたりします。プレゼン資料やマニュアルとして提出するためには、AIに「印刷用の命令」を追記させるラリーが必要です。

Larry流：PDF化用 追加プロンプト

このページをブラウザからPDFで綺麗に印刷（出力）したいです。
印刷時に背景色や画像が保持され、A4サイズで改ページが綺麗に行われるような「@media print」のCSSをコードに追加してください。

※このコードを反映させた後、ブラウザ上で Ctrl + P (Macは Cmd + P) を押し、「背景のグラフィックを印刷」にチェックを入れて「PDFとして保存」するだけです。
