Mindset

今の自分を「拡張」するスキル

AIエンジニアに転職する必要はありません。今のあなたの専門知識にAIという武器を掛け合わせることで、インプットの質を飛躍させ、可能性を劇的に広げることがLarryの目的です。

AIは「空気が読めない新入社員」

1回のプロンプトで完璧な答えを出させようとするのはやめましょう。まずは60点の叩き台を作らせ、ツッコミ（ラリー）を入れて育て上げるのがプロの思考です。

Preparation

AIのアイデアを映し出す「鏡」を用意する

プログラミングの勉強は不要ですが、AIが出力したコードをブラウザで表示させるための「プレビュー環境」は必須です。世界中のプロが使う無料の作業台「VS Code」を導入します。

1. VS Code と Live Server の導入

公式サイトからVS Codeをインストールし、拡張機能アイコンから「Live Server」を検索して追加します。

公式サイトを開く

2. 【重要】必ず「フォルダ」ごと開く

空のフォルダを作り、VS Codeの「フォルダーを開く」から読み込みます。ファイルだけを直接作るとプレビュー機能が動きません！

3. 「Go Live」で形にする

フォルダ内に index.html を作りコードを保存したら、画面右下の「Go Live」をクリックします。

Anti-Patterns

初心者が必ず引っかかる「3つのAIの罠」

AIに指示を出す際、初心者は無意識にAIを暴走させるNGプロンプトを投げてしまいます。ここでは代表的な3つの罠とその回避術を学びます。

罠①：「見た目」だけを整えさせる

NGプロンプト
「検索しやすいUI案を作って」
プロのラリー（回避策）
「実際に動かすための『ノウハウDB（裏側の仕組み）』を組み込んで作り直して」

罠②：「空気を読んで」機能を足させる

NGプロンプト
「もっと使いやすく何か良い機能を追加して」
プロのラリー（回避策）
「終わった作業をチェックできる『マイルストーン機能』を追加して」とピンポイントで指示する。

罠③：すべてを「一発」で終わらせようとする

NGプロンプト
「要件定義から実装コード、テスト手順まで全て書いて」
プロのラリー（回避策）
「まずは骨組みとなるドラフト（叩き台）だけを出力して。実装は次のステップで行います。」
Rules

意図を的確に伝える「コンテキスト（背景情報）」

AIはあなたの会社の事情を知りません。「誰が」「どんな状況で」「何のために」使うのかという背景情報を与えるだけで、出力の解像度が劇的に上がります。これはどんなツールを作る時でも必須の「お作法」です。

Context_Template.txt 
以下の要件に従って、[ 作成したいもの ] を作成してください。

【コンテキスト（前提条件）】
・私の役割：[ 例：ITリテラシーの低い社員をまとめるマネージャー ]
・ターゲット：[ 例：スマホ操作が苦手な50代の現場スタッフ ]
・目的：[ 例：日々の業務報告の心理的ハードルを下げ、提出率を上げること ]

まずは、この条件を満たすための必要な要素を箇条書きで3つ提案してください。
Rules

出力形式のコントロール術

AIは放っておくとダラダラと長い文章を書きます。「Markdown形式で」「表形式で比較して」「コードブロックのみで出力して」と、アウトプットの『器』を強制的に指定するテクニックです。無駄な前置きを削り、生産性を高めます。

比較検討させたい時

3つの案を提案し、それぞれの「メリット・デメリット・コスト」を【表形式（マークダウン）】で比較出力してください。

コードだけ欲しい時

解説や前置きは一切不要です。実装に必要なHTML/CSSのみを【1つのコードブロック】で出力してください。
Case 1: Step 1

まずは「砂場（Sandbox）」を作らせる

最初から100個の要望を詰め込まず、まずは最低限の要件で枠組みを出力させます。これが「ラリー」の第1球目です。

Prompt_1.txt
社内の日報入力ツールを作りたいです。
HTMLとCSSでシンプルな枠組みを作ってください。

必要な項目は「日付」「名前」「業務内容」の3つです。
Case 1: Step 2

「使えない箱」を実務ツールに進化させる

第1球でできた土台に対して、バリデーション（エラーチェック）やデザインフレームワークの指定を行い、実務レベルに引き上げる第2球（プロのツッコミ）です。

Prompt_2.txt
先ほどのコードは枠組みとしてはOKですが、実運用には耐えません。以下の3点を追加して作り直してください。

1. 未入力で送信した場合、赤文字でエラーを出す。
2. Tailwind CSSを用いてモダンで綺麗なデザインにする。
3. 「明日の予定」欄を追加する。
Case 1: Step 3

UXの最終調整

直前の出力の良いところを褒めて「先祖返り（せっかくのデザインが消えること）」を防ぎつつ、ユーザー目線での使い勝手（余白やクリアボタン）を向上させます。

Prompt_3.txt
素晴らしいです。最後に、UX向上のため以下の2点を微調整してください。

1. フォーム全体にもう少しゆとり（広めのPadding）を持たせてください。
2. 入力内容をリセットする「クリアボタン」を追加してください。
Case 2: LP Construction

構成案を壁打ちで作る

ランディングページ（LP）を作る際、いきなりコードを書かせると売れないゴミページが完成します。まずはプロのマーケターとして、AIと「売れる骨組み」を作ります。

Prompt_LP1.txt
あなたはプロのWebマーケターです。
[ 販売したい商品：例「AI活用オンラインスクール」 ] のLPを作成します。

いきなりコードは書かず、まずは以下の要素を含んだ「LPの構成案（ワイヤーフレーム）」をテキストで作成してください。

1. ターゲットの痛烈な悩み（3つ）
2. ヒーローセクション（キャッチコピー）
3. 解決策とベネフィット
4. 行動喚起（CTA）
Case 2: LP Construction

コピーのトーンを調教する

構成案ができたら、次は「言葉の温度感」をあなたのブランドに合わせて調整（調教）します。AI特有のお堅い文章を、人間味あふれる売れるコピーに変換します。

Prompt_LP2.txt
構成案は素晴らしいです。
ただ、現在の文章は少し「お堅い企業向け」のトーンになっています。

この構成案のすべてのテキストを、以下のトーン＆マナーで書き直してください。
・専門用語を一切使わない
・読者に「私にもできるかも！」とワクワクさせる情熱的な表現
・語尾は「〜ですよね？」「〜しましょう！」など親しみやすく
Biz & Doc

アイデアを昇華させるラリー

AIに「何かアイデア出して」と言うと、ネットに落ちている無難でつまらない答えしか返ってきません。制約や逆転の発想を与えて、思考を極限まで深掘りさせるツッコミです。

Prompt_Idea.txt
提示してくれたアイデアは少し無難すぎます。
以下のフレームワークを用いて、より尖った斬新な切り口を3つ再提案してください。

1. 【逆転の発想】業界の常識の「真逆」をいくアプローチ
2. 【極端な制約】予算が1万円しかない場合にどう達成するか
3. 【異業種ミックス】エンタメ業界の要素を取り入れたアプローチ
Biz & Doc

箇条書きから完璧な企画書へ

移動中のスマホで打った雑な箇条書きメモから、上司を説得できる論理的な企画書を一瞬で錬成する、究極の時短プロンプトです。

Prompt_Doc.txt
以下のバラバラのメモ書きを元に、A4 1枚で上司を説得できる「新規プロジェクト企画書」を作成してください。

【構成の指定】
1. 背景と課題（なぜやるのか）
2. 提案内容（何をやるのか）
3. 期待される効果（定量・定性）
4. 概算スケジュールと必要リソース

【私のメモ】
・最近、社内の情報共有が遅い
・週1のミーティングやめて、Slackの専用チャンネルで毎日10時に報告させたい
・たぶん残業代減るし、若手も発言しやすくなるはず
Biz & Doc

既存データのエラー修正（デバッグ）

自分が書いた文章の違和感や、動かないコードをAIに丸投げして「どこが悪いか」を指摘させる、あなたの専属の添削アシスタント化プロンプトです。

Prompt_Fix.txt
以下の [ 文章 / コード ] には問題（違和感やエラー）があります。

プロの視点で分析し、以下の2点を出力してください。
1. 具体的にどこが、なぜ悪いのかの指摘（箇条書き）
2. それを修正した完璧な [ 文章 / コード ]

【対象データ】
[ ここに作成途中の文章やコードを貼り付ける ]
The Output

インプットを「社内評価」に変換する

質の高い成果物を「上司やクライアントにドヤれる形（URLやPDF）」にして外へ届ける、アウトプットのスキルを学びます。今のあなたの価値を爆増させる最後のピースです。

GitHub Pages

一瞬で世界へ繋ぐ「URL化」の魔法

GitHubを使えば、ドラッグ＆ドロップだけで成果物が全世界に公開されます。「AIで作ってみたので触ってみてください」とURLをチャットで送るだけで、周りの見る目は確実に変わります。

  • 1. リポジトリ作成： GitHubで新しい箱（Public）を作る。
  • 2. ドロップ＆コミット： 作成した `index.html` などを直接放り込んで保存。
  • 3. Pagesをオンにする： 設定からBranchを「main」にしてSave。数分でURLが発行されます。
PDF Print

完璧な企画書に仕上げる「PDF調教術」

Web画面をそのまま印刷するとレイアウトが崩れます。AIに「@media print」のCSSを追加させ、印刷時専用の綺麗なレイアウトを組ませるテクニックです。

Prompt_PDF.txt
このページをブラウザからPDFで綺麗に印刷（出力）したいです。
印刷時に背景色や画像が保持され、A4サイズで改ページが綺麗に行われるような「@media print」のCSSをコードに追加してください。
Masterpiece

究極奥義：メタ・プロンプティング

あなたとAIの良い対話プロセスが完了したら、それをあなただけの「資産」にします。AI自身に「今のやり取りをテンプレート化しろ」と命じる、本スクールの総仕上げです。

Prompt_Meta.txt
今の私たちの一連の会話（ラリー）は非常に生産的でした。
このスレッドの一番最初から遡って、私があなたにどんな指示を出して、あなたがどう改善したかを分析・解析してください。

そして、初心者の同僚が次回同じ成果を出せるように、[ ] を埋めるだけで使える「汎用プロンプトのテンプレート」として出力してください。

【特典】軌道修正フレーズ集

AIの出力が微妙だった時に、サッとコピペして投げるだけでAIの軌道を元に戻せる「魔法のチートフレーズ」です。手元にストックして使い倒してください。

AIが前提を忘れた時
「一度深呼吸して、前提条件をもう一度確認してから回答を作り直してください。」
AIの回答が薄い時
「その出力は抽象的すぎます。具体的な数字や実例を交えて書き直してください。」
前置きが長すぎる時
「前置きや解説は不要です。結論（コード/文章）のみを出力してください。」
先祖返りを防ぐ時
「直前の出力は良かったです。その構成を維持したまま、以下の点だけを変更してください：」
