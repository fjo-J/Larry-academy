1-1. 脱・一発生成の罠

Mindset

今の自分を「拡張」するスキル

AIエンジニアに転職する必要はありません。今のあなたの専門知識に、AIという武器を掛け合わせることで、インプットの質を飛躍させ、可能性を劇的に広げることがLarryの目的です。

AIは「空気が読めない新入社員」

1回のプロンプトで完璧な答えを出させようとするのはやめましょう。まずは60点の叩き台を作らせ、ツッコミ（ラリー）を入れて育て上げるのがプロの思考です。

Preparation

AIのアイデアを映し出す「鏡」を用意する

プログラミングの勉強は不要ですが、AIが出力したコードをブラウザで表示させるための「プレビュー環境」は必須です。世界中のプロが使う無料の作業台「VS Code」を、小学生でも分かる手順で導入します。

1 無料ツール「VS Code」をダウンロード

まずは公式サイトからアプリをダウンロードして、パソコンにインストールします。設定はすべて「次へ」で進めてOKです。

VS Code 公式サイトを開く

2 日本語化 ＆ Live Serverの追加

VS Codeを開いたら、左側のメニューにある「ブロックのアイコン（拡張機能）」をクリックします。
検索窓に以下の2つを入力して「Install」を押してください。

① Japanese Language Pack 画面を日本語にして使いやすくします。
② Live Server コードをブラウザでリアルタイム表示するための必須ツールです。

3 【重要】必ず「フォルダ」ごと開く

デスクトップに適当な空のフォルダ（例：ai-project）を作ります。VS Codeのメニューから 「ファイル」＞「フォルダーを開く」 を選び、そのフォルダを読み込みます。

初心者の9割がここで失敗します！
フォルダを開かずに、直接「新しいファイル」を作ってしまうと、次のステップの「Go Live」ボタンが一生出現しません。

4 「Go Live」で感動の瞬間を迎える

読み込んだフォルダの中に index.html というファイルを作り、AIが書いたコードを貼り付けて保存（Ctrl+S / Cmd+S）します。
すると、VS Codeの画面の一番右下に「Go Live」という文字が出現します。これをクリックするとブラウザが立ち上がり、あなたのシステムが動き出します！

Anti-Patterns

初心者が必ず引っかかる「3つのAIの罠」

AIに指示を出す際、初心者は無意識にAIを暴走させる「NGプロンプト」を投げてしまいます。ここでは、AIが吐き出す「使えないゴミコード」や「ピント外れな回答」を防ぐための、3つの代表的な罠とその回避術（プロのラリー）を学びます。

罠①：「見た目」だけを整えさせる

初心者のNGプロンプト

「検索しやすいUI案を作って」「見やすい手順書にして」

AIは「見た目」を整えるのが得意なため、中身（実データやノウハウ）がスッカラカンな「箱だけ」を出してきます。

プロの回避策（ラリー）

「今の状態だと『ただの空の箱』になりすぎているね。実際に動かすための『裏側の仕組み（ノウハウDB）』を組み込んで作り直して。」

枠組みができた段階で「裏側の仕組みの欠落」を鋭く指摘し、中身を埋めさせます。

罠②：「空気を読んで」機能を足させる

初心者のNGプロンプト

「もっと使いやすくして」「何か良い機能を追加して」

現場のニーズに合わない、ピントのズレた一般的な機能（SNSシェア機能など）を勝手に提案してきて脱線します。

プロの回避策（ラリー）

「終わった作業をチェックできるマイルストーン機能を追加して。あと、プロンプトはコピペできるように明示して。」

AIに空気を読ませず、具体的な利用シーンを添えてピンポイントで指示します。

罠③：すべてを「一発」で終わらせようとする

初心者のNGプロンプト

「要件定義からデザイン、実装コード、テスト手順まで全て書いて」

AIの処理能力（出力文字数）の限界を超え、途中で回答が途切れたり、全体的に浅い（薄い）回答になってしまいます。

プロの回避策（ラリー）

「まずは骨組みとなるドラフト（叩き台）だけを出力して。実装コードは次のステップで指示します。」

タスクを分割し、AIの出力リソースを「1つの深い思考」に集中させます。

Rules

意図を伝える「コンテキスト（背景情報）」

AIはあなたの会社の事情を知りません。「誰が」「どんな状況で」「何のために」使うのかという背景情報（コンテキスト）を与えるだけで、出力の質は劇的に変わります。

Rules

出力形式のコントロール術

AIは放っておくとダラダラと文章を書きます。「Markdown形式で」「表形式で比較して」「コードブロックのみで出力して」と、アウトプットの『器』を指定するテクニックです。

3-1. ゼロイチ生成（第1球）

まずはシンプルなプロンプトで「砂場（Sandbox）」を作らせます。複雑な指示はまだ不要です。

3-2. プロのツッコミ（第2球）

出てきた枠組みに対して「エラーチェック（バリデーション）」や「Tailwind CSS」の指定を追加し、実務レベルに引き上げます。

3-3. UXの最終調整（第3球）

余白の調整やクリアボタンの追加など、ユーザー目線での使い勝手を向上させます。ここで前回のコードを褒めるのが先祖返りを防ぐコツです。

3-4. 構成案を壁打ちで作る (LP構築)

いきなりコードを書かせるのではなく、まずは「ターゲットの悩み」「提供価値」「行動喚起（CTA）」といったLPの骨組みをAIと一緒に作ります。

3-5. コピーのトーンを調教する

「もっと情熱的に」「専門用語を避けて小学生でもわかるように」など、文章のトーン＆マナーを自社のブランドに合わせて調教するプロンプトです。

4-1. アイデアを昇華させる

凡庸なアイデアしか出ない時、「逆転の発想で」「制約を極端に厳しくして」とAIの思考を揺さぶるラリー術です。

4-2. 箇条書きから企画書へ

頭の中にあるバラバラのメモ書きをAIに投げ、論理的なプレゼン構成に一瞬で再構築させる時間短縮テクニック。

4-3. 既存データのエラー修正

動かないコードや、違和感のある文章を丸ごと貼り付け、「どこがおかしいか指摘して直して」とデバッグさせる方法です。

The Output

インプットを「社内評価」に変換する

ここからは、質の高い成果物を「上司やクライアントにドヤれる形（URLやPDF）」にして外へ届ける、アウトプットのスキルを学びます。今のあなたの価値を爆増させる最後のピースです。

GitHub Pages

一瞬で世界へ繋ぐ「URL化」の魔法

GitHubを使えば、ドラッグ＆ドロップだけで成果物が公開されます。「AIで作ってみたので触ってみてください」とURLを送るだけで、周りの見る目は確実に変わります。

  • 1. GitHubで新しいPublicリポジトリを作る。
  • 2. 作成した `index.html` などを直接放り込んでCommit保存。
  • 3. SettingsのPagesから、Branchを「main」にしてSave。数分でURLが発行されます。
PDF Print

完璧な企画書に仕上げる「PDF調教術」

Web画面をそのまま印刷するとレイアウトが崩れます。AIに「@media printのCSSを追加して」と指示し、印刷時専用の綺麗なレイアウトを組ませるテクニックです。

6-1. メタ・プロンプティング

あなたとAIの良い対話プロセス自体を、AI自身に「マニュアル化・ノウハウ化」させる究極のスキルです。これまでのカリキュラムの集大成です。

【特典】軌道修正フレーズ集

「もっと具体的に」「別の表現で」「前提条件を忘れないで」など、コピペしてすぐ使える便利なツッコミフレーズのチートシートです。
